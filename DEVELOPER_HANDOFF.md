## Operational Notes

- For Integration API clients, use `Admin > Integration API` to create, edit display name/scopes, rotate, disable, and enable clients. The plain token is returned once on create/rotate and belongs only in the external system secret manager. Adding a scope affects the existing token immediately and requires confirmation in the UI. The client list operations panel reads existing `system_logs` to show usage/error/top-endpoint activity and can copy a scope-aware PowerShell example with a token placeholder. `npm run integration:token` remains available for controlled SQL recovery/troubleshooting only and does not write files. Emergency disable should use the UI or controlled SQL against `integration_api_clients.enabled`; if the table is removed, roll back code first.
- Before printing production Asset Labels, set `Public QR Base URL` in `Admin > Settings > Label / QR` to the permanent HTTPS domain, for example `https://asset.company.com`; the print page should show a full `/q/a/{assetId}` URL and no localhost/private warning.
- If category-prefixed suggested asset names are too long for the tape, use `Admin > Settings > Label / QR > ชื่อทรัพย์สินบน Label` and keep `ย่ออัตโนมัติ` selected. Switch to `ชื่อเต็ม` only when the label must print the exact master `Asset.name`; no separate label-name field is stored on the asset.
- For Brother PT-E550W with Laminated Tape 18mm, the app label profile is `18mm / 0.70"`. The Windows/Brother Printer Properties must also be set to Tape Size `18mm / 0.70"` before printing.
- Label text is optimized for thermal tape output with black, high-weight typography. If print quality is still faint, check Brother driver density/quality settings before changing app layout.
- On `/th/asset-management/labels`, use queue filters before selecting large batches: choose company or branch when printing a whole site/company batch, add location only when splitting a branch into smaller routes, then sort selected labels by Asset Tag, location, or category to match the physical labeling route. The "add by current filter" action follows that summarized scope. The current print-tracking model records print batches when the print page runs; a future enhancement may split "print requested" from "print confirmed" if operational proof of printer success is required.
- For mobile camera scanning, test `/th/asset-management/scan`, Serial Number scanner inputs, and `/th/audit/rounds/{id}/scan` on the target phone browser. Some browsers hide camera labels until permission is granted, so scanner helpers prefer a generic rear-camera/environment option where possible. `/th/asset-management/scan` intentionally scans Asset Label QR codes only, should default to the generic rear camera option, show an undistorted 4:3 camera preview with a smaller square white guidance frame, and should navigate to the asset detail automatically after a recognized QR scan. On supported devices, the existing preview should expose flashlight plus 1x, 2x, and 3x zoom controls; tapping 1x should return to the unzoomed preview without closing the camera, and unsupported devices should hide unavailable controls. The Asset QR camera requests high-resolution 4096x3072/30fps ideal constraints and then best-effort applies the camera track's maximum reported resolution; browsers may still choose a lower supported capture size. During UAT, center only the QR code in the frame and adjust distance until the QR modules are sharp; if the preview looks blurry, hold the label slightly farther away and use available zoom/focus tuning to enlarge the QR. Do not use the frame as a fit-box for the entire long label. `/th/audit/rounds/{id}/scan` uses the same native-resolution Asset QR decoder but stays on the audit workflow, defaults to rear-camera walking mode, continues scanning for the next label, selects the scanned asset, displays a readable Asset Tag/label in the scan input, and keeps the raw QR URL only as the latest QR reference. On supported devices, verify the existing audit camera preview exposes 1x, 2x, and 3x zoom buttons without adding a separate scan mode; tapping 1x should return to the unzoomed preview without closing the camera. Unsupported devices should hide the zoom buttons. Manual entry on Audit Scan supports partial suggestions after at least two characters and requires choosing a row when multiple matches exist; this must not change QR exact-match behavior. The sticky progress header is the only progress summary; the old metric cards are intentionally removed, and the normal camera-ready status panel stays hidden unless the camera is active, a decoded reference is available, or a camera error needs attention. After selection, verify the system data panel before choosing `บันทึกพบตรง` or `บันทึกข้อมูลไม่ตรง / Finding`; on mobile these actions are available in the bottom action bar together with `ถ่ายรูปหลักฐาน` and `สแกนต่อ / กรอกเอง`. If field data differs, use detailed mode and attach at least one evidence photo. Audit evidence photos are intentionally free-form and can be added multiple times with optional grouping tags. Use the Pending List/Zone Queue, not the successful scan result card, to mark an expected item as not found. Use the Serial Number fields when collecting manufacturer barcodes; center the barcode band on the scan line, make the bars fill most of the horizontal frame, keep fingers out of the quiet zone, and adjust distance until the vertical lines look sharp. Serial scanner crop fallback reads crop `ImageData` directly through ZXing luminance/binarizer primitives instead of passing a canvas to the browser reader, which is important on mobile browsers where the bundled reader only captures video/image elements reliably.
- Asset Detail relationship map is a read-only structural view, not the install/remove editor. It uses three lanes (`Installed under` / current item / `Has components`) plus a status summary to clarify whether the viewed asset is a parent asset, a component, both, or standalone. Relationship cards must keep the Asset Tag fully readable before the asset name and role badge; do not reintroduce one-line truncation for Asset Tags.
- Maintenance ticket creation intentionally does not require `returnDate`; `returnDate` belongs to the close-ticket workflow after repair is completed. If a damaged asset is currently `Under Inspection`, open a maintenance ticket to move it into `Pending Repair`, then progress through the maintenance workflow. If the inspection proves the asset is usable, return it to `Ready` through the controlled correction/operational workflow rather than forcing a protected status edit from the asset form.
