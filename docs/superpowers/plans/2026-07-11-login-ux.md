# Login UX Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver an accessible, password-manager-friendly Light Modern Enterprise login screen with safe return navigation and explicit session-expired feedback.

**Architecture:** Convert the route page to a small server wrapper that reads request-time search parameters and passes normalized state to a focused client form. Keep redirect/session parsing in a pure helper and keep all credential submission behavior in the client component. Reuse committed color tokens, the real app icon, next-intl, and Lucide interface icons.

**Tech Stack:** Next.js 16.2.4 App Router, React 19, TypeScript, next-auth, next-intl, Tailwind CSS 4, lucide-react, Node test runner.

## Global Constraints

- Preserve Local and AD/LDAP credential validation, the eight-hour session policy, RBAC, and role-aware `/{locale}` home routing.
- Accept callback URLs only for the active locale and same application origin.
- Keep authentication failure copy generic.
- Do not add password reset, remember me, MFA, provider selection, gradients, marketing panels, or new dependencies.
- Keep mobile inputs and the submit action at least 44px high with 16px mobile input text.
